# PPT 模板风格总结：组会第一次汇报

模板文件：`D:\Hallucination\report\ppt模板\组会第一次汇报.pptx`

分析时间：2026-05-13

## 1. 基本规格

- 页面尺寸：16:9 宽屏，约 13.33 × 7.5 英寸。
- 总页数：21 页。
- 主用途：学术组会/论文汇报。
- 整体气质：正式、克制、论文导向、校园学术风。

## 2. 视觉关键词

推荐描述给 Gemini：

> 中山大学风格的学术组会 PPT，白底为主，深绿色强调色，简洁扁平化，论文汇报气质，少量校园建筑线稿/照片，重视标题、编号、流程、表格和证据截图。

不适合的风格：

- 炫酷科技大屏风。
- 大面积渐变背景。
- 卡通插画。
- 复杂 3D 图形。
- 互联网产品发布会风。
- 暗色霓虹风。

## 3. 色彩风格

主色：

- 深绿色：`#005825`
- 相近深绿：`#00561F`

辅助色：

- 白色：`#FFFFFF`
- 深灰/黑：`#262626`、`#1F2329`
- 极浅灰背景或线稿：接近 `#F2F2F2`
- 可少量使用蓝色 `#0070C0`，但不是主色。

使用方式：

- 深绿色用于章节编号、标题强调、分割线、关键标签。
- 白底/浅灰底承载主要内容。
- 正文以深灰或黑色为主。
- 不要使用多彩饱和配色；一页中最好控制在 2-3 个主色。

## 4. 字体与字号

模板中常见字号：

- 封面/章节大标题：40-54 pt。
- 内容页标题：24-32 pt。
- 小标题/卡片标题：20-24 pt。
- 正文说明：14-16 pt。
- 注释和来源：12 pt 左右。

字体风格：

- 中文偏宋体/系统中文字体。
- 英文/数字使用默认 Office 主题字体。
- 标题可加粗，正文保持常规字重。

给 Gemini 的要求：

- 中文文字必须清晰。
- 不要使用手写体、艺术字、花体。
- 节点标签尽量短，每个标签不超过 8-10 个中文字符。

## 5. 版式规律

### 封面页

- 校园照片或校园建筑元素。
- 大标题居中或偏左。
- 作者和日期简洁放置。
- 背景不复杂，留白较多。

### 目录页

- 多个章节块，配数字编号。
- 编号使用深绿色。
- 布局整齐，有明显分组。

### 章节页

- 极简。
- 白底或浅底。
- 大号深绿色编号，如 `01`、`02`、`03`。
- 配章节标题。
- 装饰元素很少。

### 内容页

- 左上或上方标题。
- 正文常采用“标题 + 说明段落 + 图表/表格/截图”的结构。
- 常见元素：
  - 绿色小标题块。
  - 绿色编号圆点或标签。
  - 横向/纵向流程关系。
  - 论文表格、公式、折线图、柱状图截图。
  - 少量浅灰背景框。

## 6. 图片与图表风格

模板中图片主要是：

- 校园照片。
- 浅灰校园建筑线稿。
- 论文公式截图。
- 回归表格截图。
- 数据图表截图。

因此，新生成图片应贴近：

- 信息图/流程图/结构图，而不是写实插画。
- 白底、细线、深绿色强调。
- 图表元素像论文附图或报告示意图。
- 可以用简化图标，但不要过度装饰。

## 7. 用于本项目的适配建议

本项目 PPT 主题是：

> Pipeline1 / outputs / review release / ready pass 数据分析

建议沿用模板风格：

- 用深绿色作为主强调色。
- 使用“章节编号 + 短标题”的结构。
- 流程图节点用白色卡片、绿色边框或绿色标题条。
- 质量门控、review release、ready pass 可用深绿色重点突出。
- 图表页保持白底，已有统计图可直接嵌入，不要重绘成花哨风格。
- 每页只表达一个核心观点。

风格必须是学术组会 PPT：白底、深绿色 #005825 为主强调色、简洁扁平化、少量浅灰线条、无卡通人物、无大面积渐变。
标题和节点文字用中文，字数短，排版留白充足。

如果生成结果偏离模板，可继续追问：

```text
请把风格改得更像中山大学学术组会 PPT：减少装饰，白底更多，深绿色只用于编号、箭头和关键节点；删除卡通/3D/科技感元素。
```

## 9. 适合生成的五类图

1. Pipeline1 到 ready pass 的横向流程图。
2. outputs 到 ready 的 review release 质量门控图。
3. ready 数据资产地图。
4. taxonomy 与数据特征分析框架图。
5. PPT 封面/章节页背景图。

## 10. 不建议生成的图

- 复杂 dashboard。
- 虚构系统界面截图。
- 过于真实的数据中心/机器人/AI 大脑图。
- 每个数据集都写满的超密集大表。
- 与实际数据不符的伪图表。
