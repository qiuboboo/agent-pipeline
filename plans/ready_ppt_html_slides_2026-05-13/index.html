Review Release：只释放非致命风险

06 / RELEASE

Review release 不是“放水”：review 默认不进 ready，只有命中 release policy 且没有真实语义风险时才放行。

原则 1：Review 默认不放行

pass → ready
review → policy?
reject → drop

原则 2：只释放非致命工程/记录风险

  • 轻度文本-图片 alignment review
  • 图片引用链路记录不完整，但图片存在且题意可验证
  • metadata/path 指向不稳定
  • rewrite 后题干完整、答案可验证

原则 3：真实语义风险不放行

  • 关键条件缺失 / 图片缺失
  • 答案冲突 / 图文冲突
  • split variant 未确认
  • 选项缺失且答案只剩字母

第一层：结构化放行

  • exact alignment-only
  • alignment + 图片引用链路记录不完整
  • metadata/image-reference misfire
mm_math A / physreason A / ai2d A

第二层：人工候选放行

  • rewrite complete
  • answer verifiable
  • agent understandable
  • 无关键条件缺失、无答案冲突、无图文冲突
multi_physics B1/B2 / seephys A1

第三层：继续 hold/drop

  • answer conflict
  • key condition missing
  • text-image conflict
  • split unconfirmed / image recovery required
结论：review 不是单一模型识别问题，而是 alignment、metadata、视觉证据、答案标注和改写结构的混合信号。

结论：可追溯的数据资产，而不是简单拼接数据
