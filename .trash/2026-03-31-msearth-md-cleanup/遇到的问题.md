# 遇到的问题

## 1. MSEarth Open Ended 样本改写后可能丢失关键 caption 信息

### 现象
以第一个样本为例：

- 原始 query 中包含 `<image>Caption: ... Question: ...`
- rewrite 结果只保留了问题本身：
  - `Which historical periods contributed to the construction of Suoyang Ancient City?`
- 但 reasoning chain 明确引用了 caption：
  - `The caption specifically attributes its construction to the Han and Tang dynasties.`

### 风险
如果下游只消费改写后的开放题，而没有同时保留原始 caption 或相关文本资产，那么：

- 题目支撑信息可能不足
- 答案依赖的关键证据被清洗阶段删掉
- 会出现“改写后题面更干净，但语义依据更弱”的问题

### 当前观察
- 资产记录里有：
  - `question_text_source`
  - `question_text_normalized`
  - `primary_image`
- 但没有单独拆出结构化的：
  - `caption_text`
  - `image_annotation`
  - `ocr_text`
- 该样本也没有生成 `visual_structure_records`

### 待确认
- 是否应在 rewrite 时保留对 caption 的必要引用
- 是否应把 caption 从原始 query 中单独拆成独立资产/字段
- 是否需要在 open-ended variant 中显式保留 supporting context

## 2. 当前 rewrite 对已开放题样本过于保守

### 现象
这轮 MSEarth Open Ended 5 条样本全部得到：

- `strategy = keep_open`
- `rationale = Question is already open-ended.`

多数样本的 rewritten question 与 raw query 中的问题部分几乎完全一致。

### 风险
- 虽然不容易误改，但优化力度偏弱
- 对“已经勉强可用但题面不够完整”的题，改写质量提升有限
- 某些题目可能仍然缺上下文约束，导致 benchmark 风格不够稳定

### 示例
例如：
- `What influences IMERG-FCal's smaller accumulation?`

这类问题虽然是开放题，但题面偏悬空，缺少更明确的上下文约束。

### 待确认
- rewrite prompt 是否需要更积极地补足上下文
- 哪些类型的问题允许保守 keep_open，哪些需要增强式重写

## 3. MSEarth 样本 source_problem_id 目前使用的是行号，不是真实 question_id

### 现象
当前样本记录中的 `source_problem_id` 为：
- `0`
- `1`
- `2`
- ...

但数据集本身实际提供了更合适的标识字段：
- `question_id`（如 `ON019244`）

### 风险
- 不利于溯源回原始数据集
- 后续人工排查样本时不方便
- 与外部数据集 ID 对不上

### 待确认
- 是否应优先使用 `question_id` 作为 `source_problem_id`

---

## 4. reasoning_chain 已透传，但仍需确认下游是否真正消费

### 现象
目前已经新增并验证以下字段可进入主记录：
- `has_reasoning_chain`
- `reasoning_chain`

带 LLM 与不带 LLM 的 5 条样例均已跑通。

### 风险
- 当前只是“成功透传”，不等于“被后续模块有效使用”
- 可能只是写进 `problem_main_record`，但后链路并未消费

### 待确认
- 下游 Annotation / QA / 发布链路是否会读取这两个字段
- 是否需要把 reasoning_chain 拆成更结构化的节点信息
